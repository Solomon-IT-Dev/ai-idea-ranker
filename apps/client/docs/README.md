# Client Docs (Knowledge Base)

This folder contains client-specific documentation that is meant to stay stable over time:
architecture decisions, integration conventions, and development guidelines.

## Index

- `apps/client/docs/ARCHITECTURE.md` — folder layout and responsibilities
- `apps/client/docs/ROUTING.md` — route map and deep-link conventions
- `apps/client/docs/AUTH.md` — auth state, token propagation, and routing behavior
- `apps/client/docs/DEV_GUIDE.md` — development rules and patterns (networking, auth, SSE, UI)
- `apps/client/docs/API_CONTRACT.md` — API integration conventions (Swagger/OpenAPI is authoritative)
- `apps/client/docs/SSE.md` — streaming progress conventions
- `apps/client/docs/STATE_AND_DATA.md` — React Query + local state + persistence conventions
- `apps/client/docs/ERRORS.md` — error shape and UI error patterns
- `apps/client/docs/UI_GUIDELINES.md` — UI consistency guidelines
- `apps/client/docs/TROUBLESHOOTING.md` — common issues and checklists
- `apps/client/docs/ROADMAP.md` — remaining UI/UX improvements and follow-ups
